Control de acceso

En tanto que participante en el desarrollo incluyendo el maestro Scrum, el dueño del producto y los miembros del equipo de desarrollo puedo tener acceso a la aplicación para identificarse como miembro del equipo y obtener soporte al desarrollo de un producto bajo Scrum.

    • URL: '/ident/AIdentificar'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • usuario - Requerido. Long. mínima: 1.
      • clave - Requerido. Long. mínima: 1.
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. resultado:

        • label:'/VPrincipal',
        • msg:"Bienvenido usuario".

        Se continúa con el rol de duenoProducto.

      2. resultado:

        • label:'/VLogin',
        • msg:"Datos de identificación incorrectos".

    • URL: '/ident/ARegistrar'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • nombre - Requerido. Long. máxima: 50.
      • usuario - Requerido. Long. máxima: 15.
      • clave - Requerido. Long. mínima: 8. Long. máxima: 16.
      • correo - Requerido.
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. resultado:

        • label:'/VLogin',
        • msg:"Felicitaciones, Ya estás registrado en la aplicación".

      2. resultado:

        • label:'/VRegistro',
        • msg:"Error al tratar de registrarse".

    • URL: '/ident/VLogin'
    • Método: GET
    • Formato de salida: JSON
    • Roles: Sin restricción de rol

    • URL: '/ident/VPrincipal'
    • Método: GET

    • Roles: Sin restricción de rol

    • URL: '/ident/VRegistro'
    • Método: GET
    • Formato de salida: JSON

Páginas de usuarios

    • URL: '/paginas/AModificarPagina'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:

    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles
